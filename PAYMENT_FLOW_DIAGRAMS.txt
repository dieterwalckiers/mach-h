================================================================================
2. PAID EVENT COMPLETE FLOW (Happy Path)
================================================================================

1. SUBSCRIPTION FORM
   ┌─────────────────────────────────┐
   │ User fills form:                │
   │ - First name                    │
   │ - Last name                     │
   │ - Email                         │
   │ - Math CAPTCHA                  │
   └──────────────┬──────────────────┘
                  │
                  v
2. FORM SUBMISSION
   ┌─────────────────────────────────┐
   │ POST Form Data                  │
   │ → useSubscribe() action         │
   └──────────────┬──────────────────┘
                  │
                  v
3. CREATE ATTENDEE
   ┌─────────────────────────────────┐
   │ INSERT INTO attendees:          │
   │ - first_name, last_name, email  │
   │ - event_slug                    │
   │ - payment_status="pending_paym" │
   │ - payment_id=NULL (initially)   │
   └──────────────┬──────────────────┘
                  │
                  v
4. CREATE MOLLIE PAYMENT
   ┌─────────────────────────────────┐
   │ POST /api/mollie/create-payment │
   │ → Call Mollie API v2/payments   │
   │ Returns:                        │
   │ - id: tr_XXXXX                  │
   │ - status: "open"                │
   │ - checkout_url: https://...     │
   └──────────────┬──────────────────┘
                  │
                  v
5. UPDATE ATTENDEE WITH PAYMENT ID
   ┌─────────────────────────────────┐
   │ UPDATE attendees SET:           │
   │ - payment_id="tr_XXXXX"         │
   │ WHERE id={attendeeId}           │
   └──────────────┬──────────────────┘
                  │
                  v
6. REDIRECT TO MOLLIE
   ┌─────────────────────────────────┐
   │ window.location.href =          │
   │ payment.checkout_url            │
   │ (Mollie hosted checkout)        │
   └──────────────┬──────────────────┘
                  │
                  v
7. USER COMPLETES PAYMENT
   ┌─────────────────────────────────┐
   │ User enters payment details     │
   │ (Card, iDEAL, etc.)            │
   │ Payment is processed by Mollie  │
   └──────────────┬──────────────────┘
                  │
                  v
8. MOLLIE SENDS WEBHOOK
   ┌─────────────────────────────────┐
   │ POST /webhook/mollie            │
   │ body: { id: "tr_XXXXX" }        │
   │ (From Mollie servers)           │
   └──────────────┬──────────────────┘
                  │
                  v
9. WEBHOOK VERIFICATION
   ┌─────────────────────────────────┐
   │ Verify paymentId starts "tr_"   │
   │ Fetch payment details from API  │
   │ Fetch attendee from DB          │
   │ Check if already processed      │
   └──────────────┬──────────────────┘
                  │
                  v
10. CHECK PAYMENT STATUS
   ┌──────────────────────────────────┐
   │ if (payment.status === "paid")   │
   │  → Update to "confirmed"         │
   │ else if (status failure)         │
   │  → Update to "failed"            │
   └──────────────┬───────────────────┘
                  │
                  v
11. UPDATE DATABASE
   ┌──────────────────────────────────┐
   │ UPDATE attendees SET:            │
   │ - payment_status="confirmed"     │
   │ - paid_at=NOW()                  │
   │ WHERE id={attendeeId}            │
   └──────────────┬───────────────────┘
                  │
                  v
12. SEND CONFIRMATION EMAILS
   ┌──────────────────────────────────┐
   │ sendConfirmationEmails():        │
   │ 1. Internal notification         │
   │    → inschrijvingen@mach-h.be    │
   │ 2. Subscriber confirmation       │
   │    → user's email                │
   │ (Via Resend API)                 │
   └──────────────┬───────────────────┘
                  │
                  v
13. USER SEES STATUS PAGE
   ┌──────────────────────────────────┐
   │ Browser navigates to:            │
   │ /payment/status?attendeeId={id}  │
   │                                  │
   │ Initially: payment.status="paid" │
   │ DB status: "confirmed"           │
   │ Display: Green checkmark + msg   │
   │ "Bedankt voor je inschrijving!"  │
   └──────────────────────────────────┘

                    User visits page
                    /payment/status
                           |
                           v
        ┌──────────────────────────────┐
        │ usePaymentStatus() executes  │
        │ on server-side               │
        │ Fetches:                     │
        │ - Payment from Mollie        │
        │ - Attendee from DB           │
        └──────────┬───────────────────┘
                   |
                   v
        ┌──────────────────────────────┐
        │ Check payment.status         │
        └──────────────┬───────────────┘
                       |
        ┌──────────────┼──────────────┐
        v              v              v
    "pending"      "paid"         Other
      |              |              |
      v              v              v

      |         "Bedankt"       "Failed"

   Start                            v

      |

   ┌────────────────────────────┐
